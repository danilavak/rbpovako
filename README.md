# rbpovako

Сервис для рекрутинга.

## Тема

Рекрутинг: вакансии, кандидаты, отклики, интервью и офферы.

## Основные сущности

- `Vacancy` - вакансия.
- `Candidate` - кандидат.
- `Application` - отклик кандидата на вакансию.
- `Interview` - интервью по отклику.
- `Offer` - предложение кандидату.

## Основные операции

- CRUD для вакансий, кандидатов, откликов, интервью и офферов.
- Отклик кандидата на вакансию.
- Планирование интервью без пересечения слотов одного интервьюера.
- Завершение интервью с обновлением статуса отклика.
- Создание оффера после завершенного интервью.
- Принятие оффера с проверкой, что кандидат не принял другой оффер.

## База данных

По умолчанию используется H2:

```text
jdbc:h2:file:./data/rbpovako
```

Параметры подключения можно переопределить через переменные окружения:

```text
DB_URL
DB_USERNAME
DB_PASSWORD
DB_DRIVER
```

## Безопасность

API использует Spring Security:

- Пользователи хранятся в таблице `user_accounts`.
- Пароли сохраняются только как BCrypt-хеши.
- Публичная регистрация `/auth/register` создает пользователя с ролью `CANDIDATE`.
- `POST /auth/login` возвращает пару JWT access/refresh.
- `POST /auth/refresh` перевыпускает пару токенов и инвалидирует старый refresh.
- Сессии refresh-токенов хранятся в таблице `user_sessions`.
- Для API можно использовать Bearer-токен в заголовке `Authorization`.
- Basic Auth оставлен для ручной проверки и совместимости с предыдущими этапами.
- Начального администратора можно создать через переменные окружения:

```text
APP_ADMIN_USERNAME
APP_ADMIN_EMAIL
APP_ADMIN_PASSWORD
```

Роли:

- `ADMIN` - полный доступ, включая удаление.
- `HR` - работа с вакансиями, кандидатами, откликами, интервью и офферами.
- `CANDIDATE` - регистрация профиля, отклик на вакансию и принятие оффера.

JWT-настройки можно переопределить через переменные окружения:

```text
JWT_SECRET
JWT_ACCESS_TOKEN_MINUTES
JWT_REFRESH_TOKEN_DAYS
```

CSRF-токен можно получить запросом:

```http
GET /api/csrf
```
