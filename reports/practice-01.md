# Отчет по практическому заданию 1

## Тема

Система контроля версий Git и сервисы GitHub и GitLab.

## Цель работы

Создать приватные репозитории в GitHub и GitLab, подготовить локальный Git-репозиторий, настроить ветку `main`, сделать осмысленные коммиты и загрузить проект в оба удаленных репозитория по HTTPS.

## Подготовка удаленных репозиториев

Для выполнения работы были созданы два приватных репозитория:

| Сервис | Репозиторий |
| --- | --- |
| GitHub | `https://github.com/danilavak/rbpovako` |
| GitLab | `https://gitlab.com/danilavak/rbpovako` |

Для доступа к репозиториям были подготовлены токены доступа. Значения токенов в файлах проекта и в адресах удаленных репозиториев не сохранялись.

## Подготовка локального репозитория

В локальной папке проекта был создан Git-репозиторий:

```bash
git init
git branch -M main
```

На уровне репозитория были настроены имя пользователя и email:

```bash
git config user.name "Danil Vako"
git config user.email "danilavak@users.noreply.github.com"
```

Основная ветка называется:

```text
main
```

Первым коммитом был добавлен пустой `.gitignore`:

```bash
git add .gitignore
git commit -m "Добавлен пустой gitignore"
```

После этого проект был добавлен в репозиторий логическими частями через отдельные ветки и слияния.

## Настройка удаленных репозиториев

К локальному репозиторию были подключены два удаленных репозитория:

```bash
git remote add origin https://github.com/danilavak/rbpovako.git
git remote add gitlab https://gitlab.com/danilavak/rbpovako.git
```

Проверка remote:

```bash
git remote -v
```

Результат:

```text
origin  https://github.com/danilavak/rbpovako.git (fetch)
origin  https://github.com/danilavak/rbpovako.git (push)
gitlab  https://gitlab.com/danilavak/rbpovako.git (fetch)
gitlab  https://gitlab.com/danilavak/rbpovako.git (push)
```

В адресах remote не используются токены доступа.

## Загрузка проекта

Проект был отправлен в оба удаленных репозитория:

```bash
git push -u origin main
git push -u gitlab main
```

После загрузки была проверена основная ветка в обоих сервисах.

| Сервис | Ветка по умолчанию | Последний коммит |
| --- | --- | --- |
| GitHub | `main` | `7c03921` |
| GitLab | `main` | `7c03921` |

Локальный `HEAD` также указывает на коммит:

```text
7c03921
```

## Результат

В результате практического задания были созданы приватные репозитории GitHub и GitLab, настроен локальный репозиторий Git, основная ветка переименована в `main`, выполнены осмысленные коммиты на русском языке, а содержимое проекта загружено в оба удаленных репозитория по HTTPS. Токены доступа в адресах удаленных репозиториев не используются.
